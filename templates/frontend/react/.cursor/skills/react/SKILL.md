---
description: "Reactのベストプラクティスに沿ったコードを書く"
---

React（TypeScript）のベストプラクティスに従ってコードを書く。

1. コンポーネントは関数コンポーネントで定義する。`FC`型は使わず、propsの型を直接引数に定義する
2. hooksのルールを守る（ループ・条件分岐の中で呼ばない、カスタムフックは`use`で始める）
3. `useEffect`の依存配列は省略しない。依存が不要なロジックはイベントハンドラやコールバックに移す
4. `useCallback`・`useMemo`は計算コストが高い場合や子コンポーネントへの参照渡しが必要な場合にのみ使う
5. 複数コンポーネントで共有するロジックはカスタムフック（`use〇〇.ts`）に切り出す
6. stateは最小限に保つ。他の値から導出できるものはstateにしない
7. コンポーネントのpropsは必要最小限にし、データの流れを一方向に保つ
